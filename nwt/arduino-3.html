6. Variablen – Speicherplätze für Werte

Was ist eine Variable?
Eine Variable ist ein Speicherplatz im Arduino, in dem ein Wert abgelegt werden kann. Dieser Wert kann sich während des Programms ändern.
Beispiel aus dem Alltag
Stell dir eine Variable wie eine beschriftete Kiste vor:
Der Name der Kiste ist der Variablenname, der Inhalt der Kiste ist der Wert.

Der Inhalt kann jederzeit ausgetauscht werden – der Name der Kiste bleibt gleich.
Datentypen (aus dem Heft)
Variablen haben unterschiedliche Datentypen, je nachdem, was gespeichert wird:
  • int – ganze Zahlen (z. B. 10, 200)
  • long – sehr große ganze Zahlen
  • float – Kommazahlen (z. B. 2.5)
Variablen im Programm
Eine Variable wird zuerst angelegt: 
int a;
Danach kann ihr ein Wert zugewiesen werden: 
a = 10;
Später kann der Wert wieder geändert werden: 
a = a + 5;
Wichtiger Hinweis
Der Arduino merkt sich immer nur den aktuellen Wert einer Variable. Alte Werte gehen verloren, sobald ein neuer Wert zugewiesen wird.
Warum sind Variablen so wichtig?
Ohne Variablen wären Programme:
  • unflexibel
  • lang
  • schwer veränderbar
Mit Variablen kann man z. B. Tonhöhen, Wartezeiten oder Helligkeiten einfach anpassen.

denken erforderlich

  noTone(11);

Variablen machen Programme übersichtlicher und flexibler. Statt viele Zahlen im Code zu ändern, passt man nur wenige Variablen an und beeinflusst damit das gesamte Programmverhalten.

7. Serielle Ausgabe – Mit dem Arduino kommunizieren

Was ist die serielle Ausgabe?
Mit der seriellen Ausgabe kann der Arduino Text und Zahlen an den Computer senden.
Diese Informationen erscheinen im seriellen Monitor der Arduino-IDE.
Wozu braucht man das?
Die serielle Ausgabe wird vor allem genutzt, um:
  • zu prüfen, ob ein Programm richtig läuft
  • Werte von Variablen anzuzeigen
  • Fehler im Programm zu finden
Sie ist eine wichtige Hilfe beim Programmieren.
Serial.begin(9600)
Bevor der Arduino Daten senden kann, muss die serielle Verbindung gestartet werden: 
Serial.begin(9600);
Die Zahl 9600 gibt die Übertragungsgeschwindigkeit an (Baudrate).
Serial.println()
Mit Serial.println() gibt der Arduino einen Wert oder Text aus und beginnt danach eine neue Zeile. 
Serial.println(x);
So kann man verfolgen, wie sich der Wert von x im Laufe des Programms ändert.

Debugging – Wenn das Programm nicht so läuft wie gedacht

Was bedeutet Debugging?
Debugging bedeutet, Fehler im Programm zu finden und zu beheben. Fehler sind dabei normal und gehören zum Programmieren dazu.
Gute Programmierer unterscheiden sich nicht dadurch, dass sie keine Fehler machen – sondern dadurch, dass sie systematisch nach ihnen suchen.
Die wichtigste Regel
? Erst denken, dann ändern.
Wildes Ausprobieren führt selten zum Ziel. Debugging heißt: Schritt für Schritt prüfen.

Typische Fehlerquellen

Programm startet nicht
Häufige Ursachen:
  • falsches Board ausgewählt
  • falscher Port ausgewählt
  • USB-Kabel nur zum Laden geeignet
LED blinkt nicht
Mögliche Ursachen:
  • falscher Pin im Code
  • LED falsch herum eingebaut
  • Widerstand fehlt oder falsch gesteckt
Serieller Monitor zeigt nichts
Prüfe:
  • Serial.begin() vorhanden?
  • richtige Baudrate eingestellt?
  • serieller Monitor geöffnet?
Fehlermeldung auf Englisch
Die Arduino-IDE meldet Fehler meist auf Englisch. Das ist kein Problem:
Wichtig sind Schlüsselwörter wie expected, not declared, before.
Systematische Debug-Strategie
Gehe immer in dieser Reihenfolge vor:
  1. Was soll passieren?
  2. Was passiert tatsächlich?
  3. Wo im Code wird das entschieden?
  4. Welche Variable oder welcher Pin ist beteiligt?

8. for-Schleifen – Wiederholungen elegant lösen

Was ist eine for-Schleife?
Eine for-Schleife wird verwendet, wenn ein Programmteil mehrmals hintereinander ausgeführt werden soll.
Statt denselben Code immer wieder zu schreiben, nutzt man eine Schleife.
Zwar könnte man da ja auch im loop() machen, aber da wird es solange gemacht, bis das Programm gestoppt wird. Mit einer for-Schleife kann man genau festlegen, wie oft etwas passieren soll. Vielleicht nur 5 mal, oder bis eine Bedingung erfüllt ist.
Warum braucht man Schleifen?
Schleifen machen Programme:
  • kürzer
  • übersichtlicher
  • leichter veränderbar
Eine Zahl zu ändern reicht aus – der Code passt sich an.
Aufbau einer for-Schleife
Eine for-Schleife besteht aus drei Teilen: 
for (int i = 0; i < 5; i++) {
  // Code
}
Bedeutung:
  • Startwert: int i = 0
  • Bedingung: i < 5
  • Schritt: i++ (i wird um 1 erhöht)

Geschachtelte for-Schleifen – Schleifen in Schleifen

Was bedeutet „geschachtelt“?
Eine geschachtelte for-Schleife ist eine Schleife, die innerhalb einer anderen Schleife steht.
Die innere Schleife wird bei jedem Durchlauf der äußeren Schleife komplett ausgeführt.
Warum braucht man das?
Geschachtelte Schleifen werden genutzt, wenn:
  • Muster erzeugt werden sollen
  • Abläufe mehrfach wiederholt werden
  • zwei Dinge gleichzeitig gezählt werden müssen
Sie sind typisch für Licht- und Tonfolgen.
Grundaufbau 
for (int i = 0; i < 3; i++) {
  for (int j = 0; j < 5; j++) {
    // Code
  }
}
Bedeutung:
  • Die äußere Schleife läuft 3-mal
  • Bei jedem Durchlauf läuft die innere Schleife 5-mal
  • Der Code im Inneren läuft also 15-mal
Beispiel mit LED
Die LED soll in mehreren Durchgängen blinken: 
for (int r = 0; r < 3; r++) {
  for (int i = 0; i < 4; i++) {
    digitalWrite(13, HIGH);
    delay(200);
    digitalWrite(13, LOW);
    delay(200);
  }
  delay(1000);
}
? 3 Runden, in jeder Runde blinkt die LED 4-mal.
Typischer Denkfehler
Viele denken, die innere Schleife laufe „nebenbei“.
Tatsächlich gilt:
? Die innere Schleife läuft vollständig, bevor die äußere weitergeht.

    delay(100);

  }

? Beschreibe genau, was du hörst, wenn dieses Programm läuft.
Lösung anzeigen
Es erklingen jeweils 3 kurze Töne hintereinander. Diese Tonfolge wird insgesamt 2-mal wiederholt. Zwischen den beiden Durchgängen gibt es eine längere Pause.

9. LEDs dimmen – Helligkeit stufenlos steuern

Digital vs. Analog
Bisher hast du LEDs digital geschaltet:
? HIGH = an, LOW = aus.

Beim Dimmen geht es darum, die Helligkeit stufenlos zu verändern – nicht nur an oder aus.
analogWrite()
Zum Dimmen benutzt man die Anweisung: 
analogWrite(pin, wert);
Der Wert bestimmt die Helligkeit:
  • 0 ? LED aus
  • 127 ? halbe Helligkeit
  • 255 ? volle Helligkeit
Wie funktioniert das technisch?
Der Arduino erzeugt kein echtes Analogsignal, sondern nutzt Pulsweitenmodulation (PWM).

Dabei wird die LED sehr schnell ein- und ausgeschaltet. Je länger sie eingeschaltet ist, desto heller erscheint sie für das Auge.
Beispiel: LED dimmen 
void setup() {
  pinMode(3, OUTPUT);
}

void loop() {
  analogWrite(3, 127);
  delay(1000);
}
Die LED an Pin 3 leuchtet mit halber Helligkeit.
Wichtiger Hinweis
analogWrite() funktioniert nur an PWM-Pins (z. B. 3, 5, 6, 9, 10, 11 beim Arduino Uno).

10. RGB-LEDs – Farben mischen mit Licht

Was ist eine RGB-LED?
Eine RGB-LED enthält drei LEDs in einem Gehäuse: Rot, Grün und Blau.

Durch das Mischen dieser drei Farben können viele verschiedene Farben erzeugt werden.
Das RGB-Prinzip
Jede Farbe setzt sich aus Anteilen von Rot, Grün und Blau zusammen:
  • Rot + Grün ? Gelb
  • Rot + Blau ? Magenta
  • Grün + Blau ? Cyan
  • Rot + Grün + Blau ? Weiß

Ansteuerung mit dem Arduino
Jede Farbe der RGB-LED wird an einen eigenen Pin angeschlossen.
Die Helligkeit jeder Farbe wird mit analogWrite() gesteuert: 
analogWrite(rotPin, 255);
analogWrite(gruenPin, 0);
analogWrite(blauPin, 0);
? Ergebnis: Die LED leuchtet rot.
Wertebereich
Für jede Farbe gilt:
  • 0 ? Farbe aus
  • 255 ? maximale Helligkeit
Durch unterschiedliche Kombinationen entstehen neue Farben.
Gemeinsame Anode / gemeinsame Kathode
RGB-LEDs gibt es in zwei Varianten:
  • Gemeinsame Kathode: gemeinsamer Minuspol
  • Gemeinsame Anode: gemeinsamer Pluspol
Je nach Typ kann die Logik von 0 und 255 vertauscht sein.

11. if / else – Entscheidungen im Programm

Was bedeutet if?
Mit if kann ein Programm eine Entscheidung treffen.
Es prüft eine Bedingung und führt Code nur dann aus, wenn diese Bedingung erfüllt ist.
Alltagsbeispiel
Stell dir vor:
Wenn es regnet, dann nimmst du einen Regenschirm.
Sonst brauchst du keinen.

Genau so arbeitet if / else im Programm.
Grundaufbau von if / else 
if (Bedingung) {
  // wird ausgeführt, wenn die Bedingung wahr ist
} else {
  // wird ausgeführt, wenn die Bedingung falsch ist
}
Vergleichsoperatoren
Bedingungen vergleichen Werte:
  • == gleich
  • < kleiner als
  • > größer als
  • <= kleiner oder gleich
  • >= größer oder gleich
? Wichtig: == ist ein Vergleich, = ist eine Zuweisung.
