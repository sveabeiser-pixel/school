11. Eingaben mit Tastern - Der Arduino reagiert

Was ist ein Taster?
Ein Taster ist ein Schalter, der nur solange "an" ist, wie man ihn gedrückt hält.
Damit kann der Arduino auf Eingaben reagieren - ähnlich wie bei einer Tastatur.
Digitale Eingänge
Ein digitaler Eingang kennt nur zwei Zustände:
  • LOW (0) – kein Signal / nicht gedrückt
  • HIGH (1) – Signal / gedrückt
Der Arduino liest das mit digitalRead(pin).
Grundprinzip im Code
Zuerst setzt man den Pin als Eingang: 
pinMode(2, INPUT);
Dann liest man den Zustand: 
int state = digitalRead(2);
Und reagiert mit if darauf.
Pullup / Pulldown - warum braucht man das?
Ein Eingang darf nicht "in der Luft hängen", sonst kann er zufällige Werte lesen. Darum nutzt man einen Widerstand, der den Eingang eindeutig auf LOW oder HIGH zieht.

Sehr praktisch ist der interne Pullup: 
pinMode(2, INPUT_PULLUP);
Dann gilt meist:
  • Taster nicht gedrückt – HIGH
  • Taster gedrückt – LOW
Typischer Stolperstein
Bei INPUT_PULLUP ist die Logik oft "umgedreht": gedrückt = LOW, nicht gedrückt = HIGH.
Das ist kein Fehler - man muss es nur im if beachten.
Beispiel: Taster schaltet LED 
const int BTN = 2;
const int LED = 13;

void setup() {
  pinMode(LED, OUTPUT);
  pinMode(BTN, INPUT_PULLUP);
}

void loop() {
  int pressed = (digitalRead(BTN) == LOW);
  if (pressed) {
    digitalWrite(LED, HIGH);
  } else {
    digitalWrite(LED, LOW);
  }
}
Die LED leuchtet nur, wenn der Taster gedrückt wird.
Mini-Challenge - Taster als "Modus-Wechsel"
Challenge

Nutze if / else und digitalRead().
Lösung anzeigen
Mit INPUT_PULLUP gilt: gedrückt = LOW.
Also: if (digitalRead(BTN) == LOW) { grün } else { rot }
Die Farben setzt du über analogWrite() auf die RGB-Pins.

13. Wiederholungen mit while - Solange bis ...

Was ist eine while-Schleife?
Eine while-Schleife wiederholt einen Programmteil so lange, wie eine Bedingung wahr ist.

Merksatz: while = solange
Grundaufbau 
while (Bedingung) {
  // Code wird wiederholt
}
Wenn die Bedingung irgendwann false wird, endet die Schleife und das Programm läuft danach weiter.
Wann ist while sinnvoll?
Eine while-Schleife ist besonders sinnvoll, wenn man nicht vorher weiß, wie oft etwas wiederholt werden muss.

Beispiele:
  • Solange warten, bis ein Taster gedrückt wird
  • Solange blinken, bis eine Bedingung erfüllt ist
  • Solange Werte ausgeben, bis ein Grenzwert erreicht ist
Vergleich: for vs. while
for nutzt man meist, wenn die Anzahl der Wiederholungen fest ist (z. B. "5-mal blinken").

while nutzt man meist, wenn es heißt: "Solange bis ..." (z. B. "solange warten, bis gedrückt").
Achtung: Endlosschleife
Wenn sich die Bedingung nie ändert, läuft die while-Schleife endlos. Das ist einer der häufigsten Fehler.

Prüfe immer:
  • Wird eine Variable in der Schleife verändert?
  • Kann die Bedingung überhaupt falsch werden?
Beispiel 1: Zählen bis ein Grenzwert erreicht ist 
int x = 0;

void loop() {
  while (x < 20) {
    Serial.println(x);
    x = x + 5;
  }
}
Der Code läuft so lange, bis x mindestens 20 ist.
Beispiel 2: Warten auf einen Tastendruck 
const int BTN = 2;

void setup() {
  pinMode(BTN, INPUT_PULLUP);
}

void loop() {
  while (digitalRead(BTN) == HIGH) {
    // solange nicht gedrückt (HIGH)
  }
  // hier geht es weiter, sobald gedrückt (LOW)
}
Mit INPUT_PULLUP gilt: gedrückt = LOW.
Mini-Challenge - Stoppe das Blinken mit einem Taster
Challenge
Ziel: Eine LED soll blinken, solange der Taster nicht gedrückt ist. Sobald der Taster gedrückt wird, soll das Blinken stoppen und die LED aus bleiben.

Tipp: Nutze while und INPUT_PULLUP.
Lösung anzeigen
Mit INPUT_PULLUP gilt: nicht gedrückt = HIGH, gedrückt = LOW.
Daher: while (digitalRead(BTN) == HIGH) { blinken }
Danach LED ausschalten.

13. Programme planen - Denken vor dem Programmieren

Warum Programme planen?
Gute Programme entstehen nicht durch Zufall.
Bevor man Code schreibt, sollte man sich überlegen:
  • Was soll das Programm tun?
  • In welcher Reihenfolge passieren die Schritte?
  • Wo gibt es Entscheidungen oder Wiederholungen?
Genau dabei helfen Struktogramme.
Was ist ein Struktogramm?
Ein Struktogramm (auch Nassi–Shneiderman-Diagramm) ist eine grafische Darstellung eines Programms.

Es zeigt:
  • die Abfolge von Anweisungen
  • Bedingungen (if / else)
  • Wiederholungen (for / while)
– ohne eine einzige Zeile echten Codes.

Grundidee eines Struktogramms
Ein Struktogramm besteht aus Rechtecken, die von oben nach unten gelesen werden.

Wichtig:
  • Kein Pfeilchaos
  • Keine Sprünge
  • Klare Struktur
Die Leserichtung ist immer: oben – unten.
Sequenz (Abfolge)
Die einfachste Form: mehrere Anweisungen hintereinander. 
LED einschalten
warten
LED ausschalten
Jede Anweisung wird genau einmal ausgeführt – in der angegebenen Reihenfolge.
Auswahl (if / else)
Eine Entscheidung wird als geteiltes Feld dargestellt: 
Ist der Taster gedrückt?
  Ja – LED an
  Nein – LED aus
Genau ein Weg wird ausgeführt – abhängig von der Bedingung.
Wiederholung (for / while)
Wiederholungen werden als Block dargestellt, der mehrfach durchlaufen wird: 
Solange Bedingung wahr:
  LED ein
  warten
  LED aus
Das entspricht direkt einer for- oder while-Schleife.
Wichtiger Unterschied zu Flussdiagrammen
Struktogramme:
  • haben keine Pfeile
  • erzwingen saubere Struktur
  • passen direkt zu strukturiertem Code
Deshalb sind sie besonders gut für Einsteiger geeignet.

Vom Struktogramm zum Arduino-Code

Warum diese Brücke wichtig ist
Ein Struktogramm ist ein Plan, kein fertiges Programm.
Ziel ist es, diesen Plan systematisch in Code zu übersetzen, ohne etwas zu vergessen oder durcheinanderzubringen.

– Jeder Block im Struktogramm entspricht später einem bestimmten Teil im Code.
Grundregel der Übersetzung
Struktogramm von oben nach unten lesen
und jeden Block nacheinander in Code umsetzen.

Merksatz:
Ein Block = eine Idee = ein Code-Abschnitt
1. Sequenz – normaler Code
Struktogramm (gedanklich): 
LED an
warten
LED aus
Arduino-Code: 
digitalWrite(13, HIGH);
delay(500);
digitalWrite(13, LOW);
Jede Anweisung wird genau einmal und in derselben Reihenfolge ausgeführt.
2. Auswahl – if / else
Struktogramm (gedanklich): 
Ist Taster gedrückt?
  Ja – LED an
  Nein – LED aus
Arduino-Code: 
if (digitalRead(BTN) == LOW) {
  digitalWrite(LED, HIGH);
} else {
  digitalWrite(LED, LOW);
}
Der geteilte Block im Struktogramm wird zu if und else.
3. Wiederholung (fest) – for-Schleife
Struktogramm (gedanklich): 
Wiederhole 3-mal:
  LED an
  warten
  LED aus
Arduino-Code: 
for (int i = 0; i < 3; i++) {
  digitalWrite(13, HIGH);
  delay(300);
  digitalWrite(13, LOW);
  delay(300);
}
Die Anzahl der Wiederholungen steht direkt in der Schleife.
4. Wiederholung (Bedingung) – while-Schleife
Struktogramm (gedanklich): 
Solange Taster nicht gedrückt:
  LED blinkt
Arduino-Code: 
while (digitalRead(BTN) == HIGH) {
  digitalWrite(LED, HIGH);
  delay(200);
  digitalWrite(LED, LOW);
  delay(200);
}
"Solange" im Struktogramm bedeutet fast immer while.
Übersetzungs-Strategie (sehr wichtig)
Gehe immer in dieser Reihenfolge vor:
  1. Struktogramm vollständig verstehen
  2. Sequenzen umsetzen
  3. Dann Entscheidungen (if / else)
  4. Zum Schluss Wiederholungen (for / while)
So vermeidest du typische Anfängerfehler.

14. LCD-Displays - Texte anzeigen

Wozu ein Display?
Ein Display macht ein Arduino-Projekt "sichtbar": Du kannst Werte, Zustände, Menüs oder Hinweise anzeigen (z. B. Temperatur, Punktestand, Ampelmodus, Fehlermeldungen).
Es gibt verschiedene Display-Typen
Nicht jedes Display funktioniert gleich. Häufige Typen:
  • Zeichen-LCD (z. B. 16x2, 20x4; meist HD44780-kompatibel)
  • Zeichen-LCD mit I2C-Adapter (weniger Kabel)
  • OLED (oft I2C oder SPI; sehr kontrastreich)
  • TFT (Farbdisplay; oft SPI; komplexer, mehr Speicherbedarf)
In diesem Baustein geht es vor allem um das Zeichen-LCD.
Zeichen-LCD (16x2 / 20x4) - was bedeutet das?
Ein 16x2-Display kann 16 Zeichen pro Zeile und hat 2 Zeilen. Ein 20x4-Display kann 20 Zeichen pro Zeile und hat 4 Zeilen.

Es zeigt vor allem Text (Zeichen) an – keine frei zeichnbaren Bilder wie ein TFT.
Parallel vs. I2C
Es gibt zwei sehr verbreitete Anschlussarten:
  • Parallel (viele Kabel): mehrere Datenpins + Steuerpins
  • I2C (wenige Kabel): meist nur VCC, GND, SDA, SCL
Für Anfänger ist ein I2C-LCD oft einfacher, weil man weniger verdrahten muss.
Wichtig: Spannung, Kontrast, Hintergrundbeleuchtung
Häufige Stolpersteine:
  • Kontrast: Viele LCDs haben ein kleines Potentiometer – wenn der Kontrast falsch ist, siehst du "nichts".
  • 5V vs. 3.3V: Manche Displays sind 5V-tolerant, andere nicht (besonders bei OLED/TFT).
  • Backlight: Hintergrundbeleuchtung kann an/aus sein – bei falscher Verdrahtung wirkt das Display "tot".
Grundbefehle (Zeichen-LCD)
Typische Funktionen (je nach Library):
  • begin() – Display starten
  • clear() – Bildschirm leeren
  • setCursor(spalte, zeile) – Schreibposition setzen
  • print("Text") – Text ausgeben
Beispiel: 16x2 LCD (parallel, LiquidCrystal) 
#include <LiquidCrystal.h>

// Beispiel-Pins (passen ggf. an deine Verdrahtung an)
LiquidCrystal lcd(12, 11, 5, 4, 3, 2);

void setup() {
  lcd.begin(16, 2);
  lcd.print("Hallo!");
  lcd.setCursor(0, 1);
  lcd.print("Arduino LCD");
}

void loop() {
}
Hinweis: Bei Parallel-LCDs hängen die Pins von deiner Verkabelung ab.
Beispiel: LCD mit I2C (weniger Kabel) 
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

// Adresse ist oft 0x27 oder 0x3F (je nach Modul)
LiquidCrystal_I2C lcd(0x27, 16, 2);

void setup() {
  lcd.init();
  lcd.backlight();
  lcd.setCursor(0, 0);
  lcd.print("I2C LCD ready");
}

void loop() {
}
Wenn nichts erscheint: Kontrast prüfen und ggf. I2C-Adresse testen.

15. Motoren steuern

Was ist ein Motor?
Ein Motor wandelt elektrische Energie in Bewegung um.
Mit Motoren können z. B. Räder angetrieben, Arme bewegt oder Lüfter betrieben werden.

Motoren sind damit eine wichtige Verbindung zwischen Programm und realer Bewegung.
Es gibt verschiedene Motorarten
Je nach Aufgabe nutzt man unterschiedliche Motoren:
  • Gleichstrommotor (DC) - dreht sich kontinuierlich
  • Servomotor - fährt gezielt eine Position an
  • Schrittmotor - bewegt sich in festen Schritten
In diesem Lernbaustein steht vor allem der DC-Motor im Fokus.
Warum darf man Motoren nicht direkt anschließen?
Motoren benötigen meist:
  • mehr Strom als ein Arduino-Pin liefern kann
  • eine Schutzbeschaltung gegen Spannungsspitzen

Deshalb steuert der Arduino den Motor nicht direkt, sondern über ein Hilfsbauteil (z. B. Transistor oder Motortreiber).
Transistor oder Motortreiber
Häufige Lösungen:
  • Transistor: Ein/aus, evtl. mit PWM für Geschwindigkeit
  • Motortreiber-Modul (z. B. L298N): Richtung + Geschwindigkeit
Der Arduino gibt nur Steuersignale, die eigentliche Motorleistung kommt aus einer separaten Versorgung.
Wichtiger Schutz: Freilaufdiode
Motoren sind Spulen. Beim Abschalten können Spannungsspitzen entstehen, die den Arduino zerstören können.

Eine Freilaufdiode schützt die Schaltung vor diesen Spannungsspitzen.
Motor ein- und ausschalten (Prinzip)
Der Arduino steuert den Motor ähnlich wie eine LED – aber indirekt: 
digitalWrite(MOTOR, HIGH);  // Motor an
digitalWrite(MOTOR, LOW);   // Motor aus
Intern schaltet dabei ein Transistor oder Treiber den Motorstrom.
Geschwindigkeit steuern
Die Drehzahl eines DC-Motors kann mit Pulsweitenmodulation (PWM) verändert werden: 
analogWrite(MOTOR, 120);
Je größer der Wert (0–255), desto schneller dreht sich der Motor.
